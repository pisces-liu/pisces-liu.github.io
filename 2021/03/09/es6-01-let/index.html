小思考：

在定义变量的时候写 var 和不写 var 会有什么样子的区别呢？ 变量 a 和 b 有什么区别？

当使用 var 变量声明一个变量时，相当于是在当前的作用域内声明了一个变量。如果使用 var 声明的变量是在一个方法里面声明的，那么它就是一个局部变量，如果它是在全局里声明的，那么它就是一个全局变量。

而不使用 var 的话，相当于是对一个属性进行赋值。相当于在全局对象 window 上面定义一个属性 b，然后对属性 b 进行赋值。

怎样判断一个变量是否是全局对象 window 的属性呢？

JS 里面有个函数，叫做 delete 。其作用是删除对象上面的属性。delete 只能删除对象上面的某一个属性，而不能删除一个变量。

我们可以使用 delete 对 a 和b 进行删除，来判断到底哪个是变量，哪个是对象的属性。

当我们的 JS 项目变得很大的时候，我们可能会多次使用 var 进行声明变量，这样的话，所有使用 var 声明的变量都会挂载到全局对象 window 上面。这个时候，window 对象就会变得越来越大，我们将其称为污染全局变量。

let 的出现就很好的解决了这个问题。

思考题:

为什么会输出 三个 3 呢？虽然setTimeout 函数的第二个参数的时间设为了0，但是 setTimeout 是一个异步的操作，它并不会马上执行，在 JS 之中，有个叫事件循环机制（event loop）。setTimeout 这类的异步操作会等到主线程空闲，并且设置的时间到了的时候才会执行。for 循环是同步操作，它会立即执行，等到代码执行到 setTimeout 的时候，for 循环已经执行完成了，这个时候取到的值是循环以后的值，此时 i 的值已经全部变为 3 了。

现在的要求是让输出的结果是 0，1，2。怎么做？

1. 使用闭包

闭包：有一个外部函数，又有一个内部函数，内部函数会调用外部函数的变量，这样就会保证外部函数的变量的状态不会被释放。
